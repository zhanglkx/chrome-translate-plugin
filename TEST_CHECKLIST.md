## 🔧 重新加载插件步骤

### 步骤 1: 打开插件管理页面
```
1. 打开 Chrome 浏览器
2. 输入地址: chrome://extensions/
3. 或菜单 → 更多工具 → 扩展程序
```

### 步骤 2: 重新加载插件
```
找到 "ARI 划词翻译" 插件，点击刷新按钮 (↻)
或者先切换到禁用，再切换回启用
```

### 步骤 3: 打开开发者工具
```
任意网页上 → 按 F12 (或 Cmd+Option+I on Mac)
选择 "Console" 标签来查看日志
```

## 🧪 测试流程

### 测试 1️⃣: 基础功能测试
```
[ ] 打开任何网站 (如 Google, Wikipedia 等)
[ ] 选中任意英文文本 (如 "Hello world")
[ ] 应该在 2-3 秒内显示翻译弹窗
[ ] 弹窗应显示中文翻译结果
```

### 测试 2️⃣: 查看控制台日志
```
在 Console 中应该看到类似这样的日志:

📝 用户选中文本: [你选中的文本]
📨 background 收到消息: TRANSLATE_REQUEST 文本: [文本]
⏳ 开始翻译处理...
[选择一种情况]
  ✓ 词典中找到: [离线翻译]
  或 📞 调用 DeepSeek API...
✓ 翻译完成: [结果]
📤 sendResponse 发送: {...}
📥 background 响应: {...}
✓ 成功显示翻译
```

### 测试 3️⃣: 错误情况测试

#### 场景 3a: API Key 错误
```
[ ] 打开插件设置 (点击插件图标)
[ ] 输入一个错误的 API Key
[ ] 点击"保存设置"
[ ] 选中网页文本进行翻译
[ ] 应显示: ❌ 翻译失败 或 ❌ 返回格式错误
[ ] 查看 Console 应有错误日志
```

#### 场景 3b: 网络断开
```
[ ] 在"设置"中禁用"启用词典" (这样会强制使用 API)
[ ] 打开浏览器开发者工具的 Network 标签，模拟离线
[ ] 选中文本翻译
[ ] 应显示错误信息而不是卡住或无反应
```

### 测试 4️⃣: 设置功能测试
```
[ ] 点击插件图标打开设置
[ ] 输入有效的 API Key，点击保存
[ ] 测试翻译，应该成功
[ ] 禁用"启用词典"，点击保存  
[ ] 选中一个短单词（如 "cat"），应强制调用 API
[ ] 启用"启用缓存"
[ ] 选中之前翻译过的文本，应使用缓存（更快）
```

### 测试 5️⃣: 测试按钮测试
```
[ ] 打开插件设置
[ ] 输入测试文本 "Hello world"

[ ] 应立即显示结果（约 1-2 秒）
[ ] 应显示格式: ✓ 测试成功！
    翻译: [中文]

### 问题: 没有翻译弹窗出现
```

[ ] 插件是否已启用? (chrome://extensions/)
[ ] API Key 是否正确设置? (打开设置查看)

[ ] 选中的是英文还是中文? (试试英文)
```

### 问题: Console 中有 404 或权限错误
```
表示: API Key 可能无效或服务不可用

[ ] 检查 API Key 是否正确
[ ] 尝试在 popup.js 的测试按钮重新生成 API Key
[ ] 检查 deepSeek API 文档更新
